<h1>Contributing to PingElo</h1>

<p>Thank you for your interest in contributing to PingElo.
Contributions are welcome and appreciated.</p>

<h3>Ways to Contribute:</h3>
<ul>
  <li>Report bugs</li>
  <li>Suggest features</li>
  <li>Improve documentation</li>
  <li>Submit pull requests</li>
</ul>

<h3>Development Setup:</h3>
<ul>
  <li>Fork the repository</li>
  <li>Clone your fork</li>
  <li>Create a feature branch</li>
</ul>
<p>Use git checkout -b feature/your-feature-name</p>

<h3>Pull Request Guidelines:</h3>
<ul>
  <li>Keep PRs focused and small</li>
  <li>Follow existing project structure</li>
  <li>Use clear, descriptive commit messages</li>
  <li>Include comments where logic is non-obvious</li>
  <li>Ensure services still build and run</li>
</ul>

<h3>Reporting Bugs - Include:</h3>
<ul>
  <li>Steps to reproduce</li>
  <li>Expected behavior</li>
  <li>Actual behavior</li>
  <li>Relevant logs or screenshots</li>
</ul>

<h2>Questions or Ideas?</h2>
<p>Open a GitHub Issue. Discussion is encouraged.</p>
